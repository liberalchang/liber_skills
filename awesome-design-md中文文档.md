## 📋 什么是 DESIGN.md？

**DESIGN.md** 是由 Google Stitch 引入的全新概念：一份纯文本设计系统文档，供 AI 代理读取以生成风格一致的 UI。

```
✅ 只是一个 Markdown 文件
✅ 无需 Figma 导出、无需 JSON 结构、无需特殊工具
✅ 放入项目根目录，任何 AI 编码代理或 Google Stitch 都能立即理解你的 UI 风格
✅ Markdown 是大语言模型（LLM）最易读的格式，无需解析或配置
```

| 文件 | 读取者 | 定义内容 |
|------|--------|---------|
| `AGENTS.md` | 编码代理 | 如何构建项目 |
| `DESIGN.md` | 设计代理 | 项目应该呈现怎样的外观与感受 |

> 💡 **本仓库提供从真实网站提取的、开箱即用的 DESIGN.md 文件**

## 📦 每个 DESIGN.md 包含什么？

所有文件均遵循 Stitch DESIGN.md 格式，并扩展以下章节：

| 序号 | 章节 | 内容说明 |
|:---:|------|---------|
| 1 | 视觉主题与氛围 | 情绪基调、信息密度、设计哲学 |
| 2 | 色彩体系与语义 | 语义名称 + 十六进制值 + 功能角色 |
| 3 | 字体规范 | 字族定义 + 完整层级表格 |
| 4 | 组件样式 | 按钮/卡片/输入框/导航栏的多状态定义 |
| 5 | 布局原则 | 间距刻度、栅格系统、留白哲学 |
| 6 | 深度与层级 | 阴影系统、表面层级关系 |
| 7 | 设计守则 | 设计边界与反模式清单 |
| 8 | 响应式行为 | 断点定义、触控目标、折叠策略 |
| 9 | 代理提示指南 | 快速色彩参考 + 即用型提示词 |

### 每个站点包含的文件

| 文件 | 用途 |
|------|------|
| `DESIGN.md` | 设计系统文档（供 AI 代理读取） |
| `preview.html` | 视觉目录：展示色板、字号、按钮、卡片等 |
| `preview-dark.html` | 同上，但使用深色背景预览 |

## 🚀 如何使用？

```bash
# 1️⃣ 复制目标站点的 DESIGN.md 到你的项目根目录
cp path/to/awesome-design-md/claude/DESIGN.md ./DESIGN.md

# 2️⃣ 告诉你的 AI 代理使用它
# 示例提示词：
"请根据项目根目录的 DESIGN.md 文件，为我构建一个符合该设计规范的登录页面"
```

### 💡 最佳实践

```markdown
## 在提示词中明确引用 DESIGN.md

✅ 推荐：
"参考 DESIGN.md 中的色彩体系和组件规范，创建一个用户仪表盘"

✅ 进阶：
"使用 DESIGN.md 定义的按钮样式（主按钮：#007AFF，圆角 8px，悬停阴影），生成注册表单"

❌ 避免：
"做一个好看的页面" → 太模糊，无法复用 DESIGN.md 规范
```
